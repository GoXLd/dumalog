# Пример: AGENTS.md для конкретного блога

Это реальный пример блог-специфичных инструкций (Jekyll + Chirpy, франкоязычный
блог). Универсальная версия зашита в `prompts/write-post.md` и подставляет
образцовый пост автоматически — но если агент (Codex и др.) работает прямо
в репозитории блога, такой AGENTS.md в корне репо задаёт правила без dumalog.
Не забудьте добавить `AGENTS.md` в `exclude:` в `_config.yml`, иначе Jekyll
опубликует его на сайте.

# AGENTS.md — блог vande.fr (goxld.github.io)

Инструкции для агента, пишущего посты в этот блог. Эталон качества:
`_posts/2026-07-10-trois-jours-avec-claude-fable-5.md` — перед написанием
нового поста **прочитай его целиком** как образец структуры, тона и форматирования.

## Что это за репо

Jekyll + тема **Chirpy**, язык сайта французский (`lang: fr-FR`), автор GoXLd.
Push в `main` = публикация на https://vande.fr. Посты — в `_posts/`,
картинки — в `img/<тема>/`.

## Жёсткие правила (не обсуждаются)

1. **Язык поста — французский.** Технические термины, имена файлов, код — как есть.
2. **Ни одной выдуманной цифры.** Каждое число (строки кода, миллисекунды,
   мегабайты, количество коммитов) берётся из источника: логов сессий,
   memPalace, git-истории, замеров. Нет данных — не пиши число, пиши качественно
   или задай вопрос человеку.
3. **Анонимизация.** Никаких токенов, IP, доменов внутренней инфры, имён баз,
   реальных ключей на скриншотах. Проекты можно называть по типу
   («mon backend Express», «mon dashboard React»).
4. **Черновик — через ветку и PR**, не прямым пушем в `main`. Человек ревьюит
   и мержит. Альтернатива для ранних набросков: `published: false` во front matter.
5. **Не переписывай старые посты** без явной просьбы.

## Имя файла и front matter

Файл: `_posts/YYYY-MM-DD-slug-court-en-francais.md`

```yaml
---
title: Titre concret avec des chiffres si possible
description: Une phrase qui vend le post — résultat + méthode + intrigue. C'est le texte du lien partagé.
date: YYYY-MM-DD
categories: [DevOps]        # une seule catégorie parmi celles déjà utilisées dans _posts/
tags: [ia, postgresql, sre] # minuscules, réutiliser les tags existants avant d'en créer
author: GoXLd
pin: false
toc: true
published: true             # false pour un brouillon
ads: false
mermaid: true               # seulement si le post contient un diagramme mermaid
media_subpath: /img/<slug>/ # seulement si le post a des images
image:
  path: cover.png           # cover 1200x630 environ, posée dans img/<slug>/
---
```

Перед выбором `categories`/`tags` посмотри существующие:
`grep -h -A1 "^categories\|^tags" _posts/*.md | sort -u`.

## Структура поста (проверенный рецепт)

1. **Hook (2–3 абзаца).** Первое лицо, конкретная боль, масштаб в цифрах.
   Что было раньше, почему это не решалось. Сразу после — callout
   `{: .prompt-info }` с итогом в одну фразу («En une phrase : …»).
2. **Le point de départ.** Таблица «зона → состояние до». Читатель должен
   почувствовать масштаб за 10 секунд.
3. **La méthode.** Какие правила/подход применялись. Правила — списком в
   callout `{: .prompt-danger }` если они жёсткие.
4. **Хронология / основная часть.** По дням или по этапам. В каждом блоке:
   что сделано → чем доказано (замер, тест, скриншот). Диаграмма mermaid —
   если есть повторяемый цикл или поток.
5. **Замеры до/после.** Таблица с колонками Avant / Après / Gain,
   числа выровнены вправо (`|---:|`). Только реальные измерения.
6. **Честная часть.** Инциденты, откаты, «виноват оказался я сам»,
   что НЕ получилось. Это то, что отличает пост от маркетинга —
   обязательный раздел.
7. **Bilan.** Итоговая таблица метрик.
8. **Вывод.** 2–3 абзаца: что это меняет, обобщение уровнем выше
   («la question n'est plus X, mais Y»). Без пафоса больше одной финальной фразы.

Не каждый пост — про трёхдневный рефакторинг: для маленькой заметки
(«одно интересное решение») достаточно hook → проблема → решение с кодом →
замер → вывод. Структуру масштабируй, правила стиля — нет.

## Форматирование Chirpy — использовать разнообразно

- Callouts (после blockquote):
  `{: .prompt-info }` итог/резюме, `{: .prompt-tip }` приём/лайфхак,
  `{: .prompt-warning }` предостережение, `{: .prompt-danger }` жёсткие правила.
  1–4 callout на пост, не больше.
- Таблицы — для любых сравнений и списков «параметр → значение».
- Код-блоки — короткие (5–15 строк), только «вкусный» фрагмент, с комментарием
  внутри, объясняющим почему это важно
  (`// Le pattern qui coûtait des secondes...`).
- Картинки: `![alt значимый](fichier.png){: .shadow }` и **сразу следующей
  строкой** курсивная подпись `*Que montre l'image et pourquoi c'est important.*`
- Mermaid `flowchart` — максимум один на пост, только если процесс циклический
  или ветвится.
- **Жирным** — ключевые цифры и главные утверждения (2–5 на раздел, не больше).

## Стиль

- Первое лицо, разговорно-инженерный тон: «Bonjour à tous !», «Ma partie
  préférée », «honnêtement, j'ai été surpris».
- Конкретика бьёт обобщение: не «beaucoup plus rapide», а «19.8 s → ~3 s (×6)».
- Каждое утверждение о результате подкреплено: замером, таблицей, скриншотом
  или ссылкой на коммит.
- Скепсис и ограничения проговариваются явно (раздел «честная часть»).
- Никакого AI-слога: без «dans le monde d'aujourd'hui», «révolutionnaire»,
  «il est important de noter», без списков из трёх прилагательных.

## Откуда брать материал

Источник фактов — рабочие сессии (Claude Code, Codex) и memPalace:

- `mempalace search "<тема>"` — семантический поиск по проиндексированным сессиям;
- git-история целевых репо (`git log --stat --since=...`) — даты, объёмы, коммиты;
- если числа расходятся между источниками — бери git/замер, не пересказ из чата.

Хороший кандидат на пост: нетривиальное решение + измеримый результат +
чему-то учит. Плохой: рутина без цифр и без урока.

## Чек-лист перед PR

- [ ] Язык французский, файл в `_posts/`, имя `YYYY-MM-DD-slug.md`
- [ ] Front matter полный, категория/теги из существующих
- [ ] Все цифры прослеживаются к источнику
- [ ] Есть «честная часть» (инцидент/ограничение/что не вышло)
- [ ] Анонимизация: нет токенов, IP, внутренних доменов, имён баз
- [ ] Картинки лежат в `img/<slug>/`, у каждой alt + подпись
- [ ] `bundle exec jekyll build` проходит без ошибок (если bundle доступен)
- [ ] Ветка + PR, в описании PR — источники фактов (сессии/коммиты)
